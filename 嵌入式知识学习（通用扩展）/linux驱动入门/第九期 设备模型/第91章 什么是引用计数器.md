# 一、引用计数器介绍：

##  什么是引用计数器。
### 1 、内存管理技术，用于跟踪对象或资源的引用数量


### 2 、在对象被引用时增加计数值，并在引用被释放时减少计数值(❤️)


### 3 、以确定何时可以安全地释放对象或资源。



### 4 、基本原理如下：(❤️)
> 对象或资源被**创建时**，引用计数器**初始化为1。**
> 
> 当**有新的引用指向对象或资源时，引用计数器增加。**
> 
> 当引用不再指向对象或资源时（引用被删除、超出作用域等），引用计数器减少。
> 
> **当引用计数器的值为0时**，表示没有任何引用指向对象或资源，可以安全地释放对象或资源，并进行相关的**清理**操作。

## 引用计数器kref介绍
### 1 、内核中提供的一种引用计数器实现


### 2 、管理内核中的对象的引用计数。


### 3 、结构体kref来表示：
- 1 定义在include/linux/kref.h头文件当中

```c
struct kref {
	refcount_t refcount;
};
 
typedef struct {
	   atomic_t refs;
} refcount_t;
 
typedef struct {
	int counter;
} atomic_t;
```

- 2 本质是一个int型变量

### 4 、将结构体kref嵌入到其他结构体中使用(❤️)
- 1 例如struct kobject，以实现引用计数的管理。

> **通过对struct kref的操作来对struct kobject进行引用计数的管理**，并在引用计数减少到0时释放相关资源。

## 常用api函数
### 1 、`kref_init(struct kref *kref)：`
> 函数作用：**初始化一个 struct kref 对象**。在使用引用计数之前，必须先调用此函数进行初始化。初始化kerf的**值为1**

```c
函数原型：
static inline void kref_init(struct kref *kref)
{
    refcount_set(&kref->refcount, 1);

}
```

### 2 、`kref_get(struct kref *kref)：`计数值加1(❤️)
> 函数作用：增加 struct kref 的引用计数。每次调用此函数，引用计数都会增加。**kref计数值加1**

```c
函数原型：

static inline void kref_get(struct kref *kref)

{

  refcount_inc(&kref->refcount);

}
```


### 3 、`kref_put(struct kref *kref, void (*release)(struct kref *))：`(❤️)
> 函数作用：**减少 struct kref 的引用计数，并在引用计数减少到零时调用 release 函数来进行资源的释放**。通常，release 函数会在其中执行对象的销毁和内存释放等操作。

```c
static inline int kref_put(struct kref *kref, void (*release)(struct kref *kref))

{

    if (refcount_dec_and_test(&kref->refcount)) {

        release(kref);

        return 1;

    }

    return 0;

}
```


### 4 、`void refcount_set(refcount_t *r, int n)`
> 函数作用：**设置kerf的计数值**。

```c
static inline void refcount_set(refcount_t *r, int n)

{

atomic_set(&r->refs, n);

}

```
