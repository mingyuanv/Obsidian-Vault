# 四、 第79章 完善drop和release函数实验

## release和drop函数的区别

>` .release`成员字段是在` struct configfs_item_operations`结构体中定义的一个回调函数指针。它指向一个函数，当 configfs 中的**配置项被释放或删除时，内核会调用该函数来执行相应的资源释放操作**。它通常用于释放与配置项相关的资源，比如<span style="background:#d3f8b6">释放动态分配的内存、关闭打开的文件描述符</span>等。

> `.drop_item` 是在 `struct configfs_group_operations`结构体中定义的一个回调函数指针。它指向一个函数，当 configfs 中的**配置组（group）被删除时**，内核会调用该函数来处理与配置组相关的操作。这个函数通常用于清理配置组的状态、释放相关的资源以及执行其他必要的清理操作。.drop_item函数在删除配置组时被调用，而不是在删除单个配置项时被调用。

> `.release`成员字段用于**配置项**的释放操作，而 `.drop_item`成员字段用于**配置组**的删除操作。它们分别在不同的上下文中执行不同的任务，但都与资源释放和清理有关。

# 六、第81章 实现多级目录实验
> 我们了解到一个配置组也就是（group）可以包含多个配置项（item）和子配置组。**配置项和配置组都是作为配置组的成员存在**的。配置项和配置组之间通过指针进行关联，以形成一个层次结构。
