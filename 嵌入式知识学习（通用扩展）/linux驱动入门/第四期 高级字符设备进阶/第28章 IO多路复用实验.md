## IO多路复用简介
### 1 、IO多路复用工作原理
> IO多路复用可以实现一个进程监视多个文件描述符。一旦某个文件描述符准备就绪，就通知应用程序进行相应的读写操作。<span style="background:#d3f8b6">没有文件描述符就绪时就会阻塞应用程序</span>，从而释放出CPU资源。
> 
### 2 、三种实现IO多路复用的模型（select、poll 和 epoll）


### 3 、三种模型区别
> **poll函数和seslect函数**都可以监听多个文件描述符，通过<span style="background:#d3f8b6">轮询</span>来获取已经准备好的文件描述符。
> 但是**epoll函数**将主动轮询变成了被动通知，当<span style="background:#d3f8b6">事件发生时被动接收通知</span>。

- 1 epoll的效率更高

> 在单个线程中，**select函数**<span style="background:#d3f8b6">最大可以监视1024</span>个文件描述符，
> 而**poll函数**和select函数并没有什么区别，只是poll函数<span style="background:#d3f8b6">没有最大</span>文件描述符的限制。

### 4 、poll函数简介 - 监视并等待多个文件描述符的属性变化
#### 函数原型：
```c
int poll(struct pollfd *fds,nfds_t nfds,int timeout);
```

#### 函数参数：
- 1 第一个参数fds: 
> - **要监视的文件描述符集合以及要监视的事件**，为一个**数组**，<span style="background:#d3f8b6">数组元素都是结构体</span>pollfd类型，pollfd结构体如下所示：

```c
struct pollfd {
    int fd;       //被监视的文件描述符
    short events;  //等待的事件
    short revents;  //实际发生的事件
}；
```
> 在pollfd结构体中，第一个成员fd是被监视的文件描述符。第二个成员events是要监视的事件，
> **第三个成员是返回事件**，由<span style="background:#d3f8b6">Linux内核设置具体的返回事件</span>。

- 2 可监视的事件类型如下所示：
> - POLLIN 有数据可以读取
> - POLLPRI 有紧急的数据需要读取
> - POLLOUT 可以写数据
> - POLLERR 指定的文件描述符发生错误
> - POLLHUP 指定的文件描述符挂起
> - POLLNVAL 无效的请求
> - POLLRDNORM 等同于POLLIN


- 1 第二个参数nfds: poll函数要监视的文件描述符数量

- 1 第三个参数timeout:指定等待的时间，单位是ms。
> - 无论I/O是否准备好，时间到POLL就会返回。如果timepoll大于0 等待指定的时间，如果timeout等于0，立即返回。如果**timeout等于-1，事件发生以后才返回**。

#### 函数返回值：
- 2 失败返回-1，成功返回revents不为0的文件描述符个数。





### 5、驱动中的poll函数
> 当应用程序使用select或者poll函数对驱动程序进行非阻塞访问时，驱动程序中**file_operations操作集的poll函数**会执行。所以需要完善驱动中的poll函数。

#### 函数原型
```c
unsigned int (*poll)(struct file *file,struct poll_table_struct *wait);
```

#### 函数参数：
> filp:要打开的**文件描述符**
> 
> wait: 结构体poll_table_struct类型指针，此参数是**由应用程序中传递**的。一般**此参数要传递给poll_wait函数**。


#### 返回值：向应用程序返回资源状态

> - POLLIN 有数据可以读取
> - POLLPRI 有紧急的数据需要读取
> - POLLOUT 可以写数据
> - POLLERR 指定的文件描述符发生错误
> - POLLHUP 指定的文件描述符挂起
> - POLLNVAL 无效的请求
> - POLLRDNORM 等同于POLLIN，普通数据可读。

#### 函数功能：
- 1 这个函数要进行下面两项工作。
> 首先，对可能引起设备文件状态变化的等待队列**调用poll_wait()**,将对应的**等待队列头添加到poll_table**.然后**返回表示是否能对设备进行无阻塞读写访问的掩码**。


### 6、poll_wait函数（poll_wait函数是不会引起阻塞的）
```c
void opll_wait(struct file *filp,wait_queue_heat_t *queue,poll_table *wait);
```

> 参数queue是要添加到poll_table中的等待队列头，参数wait是poll_table，也就是file_operations中**poll函数的wait参数**。

> -将当前应用进程（调用 `poll` 或 `select` 的进程）挂起到 `queue` 指向的等待队列中，以便在等待队列对应的事件发生时被唤醒。
> 
> - `poll_wait` 会注册一个唤醒回调函数（`poll wake`），当 `read_wq` 队列被唤醒时（例如通过 `wake_up_interruptible(&read_wq)`），回调函数会通知内核重新检查设备状态。


- 2 `poll` 函数需要返回一个掩码（mask），表示当前设备的状态（如可读、可写）


### 7、`poll_wait` 与进程阻塞的关系

> - **`poll_wait` 本身不会阻塞进程**，它只是将进程添加到等待队列中。
> - **进程的阻塞发生在用户空间的 `poll` 或 `select` 调用中**。内核会通过 `poll_table` 跟踪所有等待队列，并在超时或事件发生时返回结果。
> - 当设备驱动调用 `wake_up_interruptible(&read_wq)` 时，等待队列中的进程会被唤醒，内核会重新调用 `poll` 函数检查设备状态。
>

## 编写测试 APP
### 1 、编写好的应用程序read.c如下所示：
```c
#include <stdio.h>
#include <sys/types.h>
#include <sys/stat.h>
#include <fcntl.h>
#include <unistd.h>
#include <stdlib.h>
#include <poll.h>


int main(int argc,char *argv[])
{

    int fd;
    int ret;
    struct pollfd fds[1];

    char buf1[32]={0};
    char buf2[32]={0};

    fd=open("/dev/test",O_RDWR); //打开/dev/test设备，阻塞式访问
    if(fd<0){
        printf("open error\n");
    }

    fds[0].fd=fd;
    fds[0].events=POLLIN; //监视数据是否可以读取
    printf("read before \n");

    while(1){

        ret=poll(fds,1,3000);  //轮询文件是否可操作，超时3000ms
        if(!ret){  //超时
            printf("time out !!\n,");

        }else if(fds[0].events==POLLIN){ //如果返回事件是有数据可读取
            read(fd,buf1,sizeof(buf1));
            printf("buf1 is %s \n,",buf1); //打印读取的数据
            sleep(1);

        }

    }

    printf("read after\n");
    close(fd);

    return 0;
}

```
> 上述代码第16行，在打开设备节点时不使用非阻塞方式，要使用阻塞方式，所以改为O_RDWR。
> 在上述代码的第28行，使用poll函数监视并等待多个文件描述符的属性变化。poll函数第一个参数是被监视的文件描述符，是pollfd结构体类型的数组，所以在14行定义了pollfd结构体类型的数组fds。poll函数第2个参数是要监视的文件描述符数量，这里监视的文件描述符为1个。poll函数第3个参数是指定等待的时间 3000ms。

    {
        perror("open error \n");

}

## 驱动程序编写
### 1 、字符设备、poll、open、read、write函数、等待队列（编写好的程序）
```c
#include <linux/init.h>
#include <linux/module.h>
#include <linux/kdev_t.h>
#include <linux/fs.h>
#include <linux/cdev.h>
#include <linux/uaccess.h>
#include <linux/io.h>
#include <linux/wait.h>
#include <linux/poll.h>

struct device_test{

    dev_t dev_num;//设备号
    int major;  
    int minor;

    struct cdev cdev_test;  // 字符设备结构体
    struct class *class;    // 类指针
    struct device *device;  // 设备指针

    char kbuf[32];  // 内核缓冲区,存放数据
    int flag;  // 标志位，表示是否有数据可读

};

// 实例化一个设备结构体变量
struct device_test dev1;

// 定义并初始化等待队列头，用于阻塞读取
DECLARE_WAIT_QUEUE_HEAD(read_wq);


static int cdev_test_open(struct inode *inode,struct file *file){

    // 设置私有数据为设备结构体地址，方便其他函数使用
    file->private_data=&dev1;

    printk("This is cdev_test_open\r\n");

    return 0;
}

static int cdev_test_release(struct inode *inode, struct file *file)
{
    printk("This is cdev_test_release\r\n");
    return 0;
}


static ssize_t cdev_test_write (struct file *file,const char __user *buf,size_t size,loff_t *off){

    struct device_test *test_dev=(struct device_test *)file->private_data;

    if(copy_from_user(test_dev->kbuf,buf,size)!=0){

        printk("copy_from_user  error\r\n");

        return -1;
    }

    // 设置标志位，表示有新数据到达
    test_dev->flag=1;

    //唤醒 等待队列（read_wq）中处于 可中断睡眠状态（TASK_INTERRUPTIBLE）的进程。
    //通常用于通知等待资源（如数据、硬件状态）的进程：“资源已就绪，请继续执行”。
    wake_up_interruptible(&read_wq);

    return 0;
}


static ssize_t cdev_test_read (struct file *file, char __user *buf,size_t size,loff_t *off){

    struct device_test *test_dev=(struct device_test *)file->private_data;

    // 如果是非阻塞模式并且没有数据，则立即返回
    if(file->f_flags & O_NONBLOCK){
        if(test_dev->flag !=1){

            return -EAGAIN;
        }
    }

    // 阻塞等待直到有数据可用
    wait_event_interruptible(read_wq, test_dev->flag);



    if(copy_to_user(buf,test_dev->kbuf,strlen(test_dev->kbuf))!=0){

        printk("copy_from_user  error\r\n");

        return -1;
    }

    printk("This is cdev_test_read\r\n");

    return 0;
}


static __poll_t cdev_test_poll(struct file *file,struct poll_table_struct *p){

    struct device_test *test_dev=(struct device_test *)file->private_data;  //设置私有数据

    __poll_t mask=0;

    // 将当前进程（调用 poll 或 select 的进程）挂起到 wait_address 指向的等待队列中，以便在等待队列对应的事件发生时被唤醒。
    poll_wait(file,&read_wq,p);//应用阻塞

    // 如果有数据可读，设置 POLLIN 标志
    if(test_dev->flag==1){

        mask |= POLLIN;
    }

    return mask;
}





struct file_operations cdev_test_fops={

    .owner = THIS_MODULE, //将owner字段指向本模块，可以避免在模块的操作正在被使用时卸载该模块
    .open=cdev_test_open,
    .read=cdev_test_read,
    .write=cdev_test_write,
    .poll=cdev_test_poll,
    .release=cdev_test_release

};


static int __init chr_fops_init(void){

    int ret;

    /*1 创建设备号*/
    ret=alloc_chrdev_region(&dev1.dev_num,0,1,"alloc_name");
    if(ret<0){
        goto err_chrdev;
    }
    printk("alloc_chrdev_region is ok\n");

    //打印设备号
    dev1.major=MAJOR(dev1.dev_num);
    dev1.minor=MINOR(dev1.dev_num);
    printk("major is %d \r\n", dev1.major); //打印主设备号
    printk("minor is %d \r\n", dev1.minor); //打印次设备号

    /*2 初始化cdev*/
    dev1.cdev_test.owner=THIS_MODULE;
    cdev_init(&dev1.cdev_test,&cdev_test_fops);

    /*3 添加一个cdev,完成字符设备注册到内核*/
   ret=cdev_add(&dev1.cdev_test,dev1.dev_num,1);
    if(ret<0)
    {
        goto  err_chr_add;
    }

    /*4 创建类*/
    dev1.class=class_create(THIS_MODULE,"test");
    if(IS_ERR(dev1.class)){

        ret=PTR_ERR(dev1.class);
        goto err_class_create;
    }

    /*5  创建设备*/
    dev1.device=device_create(dev1.class,NULL,dev1.dev_num,NULL,"test");
    if(IS_ERR(dev1.device))
    {
        ret=PTR_ERR(dev1.device);
        goto err_device_create;
    }

return 0;

//错误处理
err_device_create:
    class_destroy(dev1.class);

err_class_create:
    cdev_del(&dev1.cdev_test);

err_chr_add:
    unregister_chrdev_region(dev1.dev_num,1);

err_chrdev:
    return ret;

}



static void __exit chr_fops_exit(void) //驱动出口函数
{
    /*注销字符设备*/
    unregister_chrdev_region(dev1.dev_num, 1); //注销设备号
    cdev_del(&dev1.cdev_test);                 //删除cdev
    device_destroy(dev1.class, dev1.dev_num);       //删除设备
    class_destroy(dev1.class);                 //删除类
}


module_init(chr_fops_init);
module_exit(chr_fops_exit);

MODULE_LICENSE("GPL v2");
MODULE_AUTHOR("topeet");
```
