# 一、
## 1、kmemdup();
```c
    myitem->addr = kmemdup(page,size,GFP_KERNEL);
```
> 蒋`page` 的数据复制数据到一个新的分配的内存区域，并将这个新分配的内存区域的地址赋值给 `myitem->addr`

> - **`kmemdup`**：这是一个Linux内核函数，它接受三个参数：**源数据的起始地址**、要复制的数据大小以及内存分配标志。它的功能是分配指定大小的内存，并将源数据复制到这块新分配的内存中。如果内存分配失败，函数将返回 `NULL`。
>     
> - **`GFP_KERNEL`**：这是分配标志，**指示内核如何以及何时分配内存**。`GFP_KERNEL` 表示这是一个普通的内核内存分配请求，在分配内存时可以进行睡眠等待，因此不能在中断上下文中使用此标志。
>     
> - **`myitem->addr`**：假设 `myitem` 是一个指向包含 `addr` 成员的结构体的指针。这行代码将新分配并填充了数据的内存块地址赋值给 `addr`，使得可以通过 `myitem->addr` 来访问这块内存中的数据。

## 3、kzalloc

### 除了分配内存之外还会将这块内存的所有字节设置为0


### 原型
```c
void *kzalloc(size_t size, gfp_t flags);
```
> - `size` 表示需要分配的内存大小。
> - `flags` 是内存分配的类型标志，指定如何以及从哪里分配内存



```c
    mygroup = kzalloc(sizeof(*mygroup),GFP_KERNEL);
```
> - **`GFP_KERNEL`**：这是一个标志，指示`kzalloc`应如何分配内存。`GFP_KERNEL`是最常用的标志之一，表示这次**分配是在内核上下文中进行的**（而不是中断上下文），允许分配过程休眠、交换等操作，以等待足够的内存变得可用。这意味着**调用该函数的地方不能处于原子上下文**（如中断处理程序）。

## 4、ioremap函数
### 函数原型
```c
#include <asm/io.h> // 或者更通用的 #include <linux/io.h> 
void __iomem *ioremap(phys_addr_t offset, size_t size);
```

### 参数解析
> **返回值**: `void __iomem *`：这是一个**指向映射后内核虚拟地址的指针**。

> 参数 `offset`，代表 **要映射的物理地址**。
> 第二个参数 `size`，代表 **要映射的内存区域大小**，单位是**字节**。

## 2、`strcmp()`:字符串比较

### 按字典序比较两个以空字符 `'\0'` 结尾的字符串。


### 函数原型
```c
#include <string.h> // 使用前必须包含此头文件 
int strcmp(const char *str1, const char *str2);
```

### 参数
> - `const char *str1`: 指向第一个要比较的字符串的**指针**。
> - `const char *str2`: 指向第二个要比较的字符串的指针。
> - `const` 关键字表示函数不会修改传入的字符串内容。


### 返回值: `int`
> - **返回 0**: 表示两个字符串 **完全相等**。
> - **返回负整数** (通常是 -1, -2 等): 表示 `str1` **小于** `str2`。
> - **返回正整数** (通常是 1, 2 等): 表示 `str1` **大于** `str2`。


## 3、dev_err - 设备驱动程序中的错误处理和调试
### 
> - **报告硬件操作失败​**​：例如配置引脚模式、上电时序、寄存器读写等操作失败时。
>     
> - ​**​资源申请失败​**​：如内存申请、中断请求等失败时。
>     
> - ​**​函数执行失败​**​：当某些关键函数返回错误码时，常用 `dev_err`打印错误信息并记录失败原因。
> 
### 类似的开发打印函数
| 函数               | 打印级别         | 用途                                          | 是否依赖CONFIG_DYNAMIC_DEBUG |
| ---------------- | ------------ | ------------------------------------------- | ------------------------ |
| ​**​dev_err​**​  | KERN_ERR     | 报告​**​错误​**​信息，用于硬件操作失败、资源申请失败等严重问题         | 否                        |
| ​**​dev_warn​**​ | KERN_WARNING | 报告​**​警告​**​信息，表示可能出现问题，但系统通常能继续运行          | 否                        |
| ​**​dev_info​**​ | KERN_INFO    | 输出​**​一般信息​**​，常用于设备初始化成功、探测到设备等通知性信息       | 否                        |
| ​**​dev_dbg​**​  | KERN_DEBUG   | 输出详细的​**​调试​**​信息，只有在开启动态调试或定义`DEBUG`宏时才会打印 | 是                        |
|                  |              |                                             |                          |

### 核心信息摘要

|项目|说明|
|---|---|
|​**​功能​**​|在设备驱动中打印错误信息，报告设备相关操作失败|
|​**​输出级别​**​|`KERN_ERR`(3)|
|​**​输出目标​**​|内核日志缓冲区 (可通过 `dmesg`查看)|
|​**​主要参数​**​|`struct device *dev`: 指向设备对象的指针；`const char *fmt`: 格式化字符串|
|​**​典型应用场景​**​|引脚配置、电源管理、寄存器访问、内存申请等设备操作失败时|
|​**​优势​**​|除了打印信息，还会附带设备标识信息，便于定位问题源头；是 `printk`的封装版|
