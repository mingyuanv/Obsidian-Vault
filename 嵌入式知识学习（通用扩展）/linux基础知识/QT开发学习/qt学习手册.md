# $1.1.2\mathrm{C + + }$  关键字

# 1.1.3 C++头文件

每个  $\mathtt{C + + }$  程序由头文件和定义文件组成。头文件作为一种包含功能函数、数据接口声明的载体文件，主要用于保存程序的声明，而定义文件用于保存程序的实现。

头文件是用户应用程序和函数库之间的桥梁和纽带。编译时，编译器通过头文件找到对应的函数库，进而把已引用函数的实际内容导出来代替原有函数，然后再进行翻译。

常用的头文件：

- 标准输入输出流：<iostream>- 标准文件流：<fstream>- 标准字符串处理函数：<string>- 标准数学函数：<cmath>除了可以使用系统提供的头文件外，程序员还可以定义自己的头文件。

# 1.1.5  $\mathbb{C} + +$  命名空间

命名空间是  $\mathtt{C + + }$  中用于组织和管理标识符（如变量、函数、类等）的一种机制，以避免命名冲突。命名空间提供了一种将标识符分组的方式，以便在不同的上下文中使用相同的标识符而不会发生冲突。

$\mathtt{C + + }$  中有标准库的命名空间std，包含了许多  $\mathtt{C + + }$  标准库的函数和类。为了使用std命名空间中的成员，你可以在代码中使用using namespace std；或者在需要使用的标识符前加上std：前缀。后续如果有未指定命名空间的符号，那么默认使用std，像常用的cin、cout、cerr、endl等都位于命名空间std。

注：在实际项目中，最好避免在全局范围内使用using namespace std；，以减少潜在的命名冲突。最好选择在需要使用的具体位置添加std：前缀

定义命名空间，需要使用到namespace关键字，后面跟命名空间的名字，然后接一对即可，中即为命名空间的成员。

命名空间中的内容，既可以定义变量，也可以定义函数。

# 1.3.4 类的继承

什么是类的继承？

# 1.4 函数的重载

函数重载是  $\mathtt{C + + }$  中的一种特性，它允许你在同一个作用域内定义多个同名函数，但这些函数具有不同的参数列表。在调用重载函数时，编译器会根据传递的参数类型或个数来确定调用哪个函数。函数重载有助于提高代码的可读性和灵活性，因为你可以使用相同的函数名来执行不同的操作。

# 1.5构造函数和析构函数

构造函数：用于初始化对象的状态。当创建类的对象时，构造函数会自动调用，并设置对象的初始属性。构造函数的名称与类名相同，没有返回类型（甚至没有void）。可以定义多个构造函数，称为重载构造函数，根据传递的参数来选择调用哪个构造函数。

包含纯虚函数的类被称为抽象类。抽象类不能被实例化，只能被用作其他类的基类。派生类必须实现纯虚函数，否则它们也会成为抽象类（如果派生类没有提供实现，编译器将发出错误）。纯虚函数常常用于定义接口，让多个派生类共享相同的接口规范，但各自提供不同的实现。示例：

# 3.1.1布局管理Layouts

Qt提供了多种布局管理器（Layouts），用于实现不同的布局需求。如下图所示：

![](assets/qt学习手册/4890f782ede22e41440d67b1acbac06f21ee9cc6eb1fe880e6915ab2a4c68127.jpg)

1. QHBoxLayout（水平布局）和QvBoxLayout（垂直布局）：

$\bullet$  这两种布局管理器用于在水平和垂直方向上排列窗口部件。 $\bullet$  可以通过在布局中添加窗口部件来按照顺序排列它们。 $\bullet$  常用于创建简单的工具栏、状态栏、按钮栏等。

2. QGridLayout（网格布局）：

$\bullet$  网格布局将窗口部件排列成网格形式，每个窗口部件占据一个格子。 $\bullet$  可以使用行号和列号来指定窗口部件的位置。 $\bullet$  常用于创建复杂的表单和网格布局。

3. QFormLayout（表单布局）：

$\bullet$  表单布局用于创建简单的表单，通常由标签和输入字段组成。 $\bullet$  自动对齐标签和输入字段，使其呈现良好的外观。 $\bullet$  常用于创建设置对话框和表单输入界面。

2. Vertical Spacer:垂直间隔

QSpacerItem 继承 QLayoutItem。它允许你在布局中插入一个可调整大小的间隔。可以在水平和垂直方向上插入间隔项，以控制控件之间的距离。可以设置间隔项的大小、最小大小、最大大小等属性。

# 3.1.3 按钮 Buttons

在 Qt 里，最常用使用的控件就是按钮了，有了按钮，我们就可以点击，从而响应事件，达到人机交互的效果。不管是嵌入式或者 PC 端，界面交互，少不了按钮。

Qt 按钮部件是一种常用的部件之一，Qt 内置了六种按钮部件如下：

<table><tr><td>Buttons</td></tr><tr><td>Push Button</td></tr><tr><td>Tool Button</td></tr><tr><td>Radio Button</td></tr><tr><td>Check Box</td></tr><tr><td>Command Link Button</td></tr><tr><td>Dialog Button Box</td></tr></table>

1. QPushButton（推按钮）：

$\bullet$  用于触发操作，如保存文件、提交表单等。 $\bullet$  它可以包含文本和/或图标，并且通常用于执行用户交互操作。 $\bullet$  可以连接到槽函数以处理按钮点击事件。

2. QToolButton（工具按钮）：

$\bullet$  用于工具栏和工具栏类应用程序的特殊按钮。 $\bullet$  它可以包含文本、图标和下拉菜单，并且提供多种显示模式。 $\bullet$  通常用于创建工具栏按钮、弹出菜单按钮等。

3. QRadioButton（单选按钮）：

$\bullet$  用于创建单选按钮组，允许用户从一组选项中选择一个。 $\bullet$  多个单选按钮可以组合在一起，但只能选择一个。 $\bullet$  常用于选项卡、性别选择、主题切换等场景。

4. QCheckBox（复选框）：

$\bullet$  用于创建复选框，允许用户选择多个选项。

- 多个复选框可以独立使用，用户可以选择一个或多个。- 常用于启用/禁用选项、多项选择等场景。

5. QCommandLinkButton（命令链接按钮）：

$\bullet$  是一种特殊的按钮，通常用于显示带有描述性文本的重要命令。 $\bullet$  它具有大而突出的外观，用于吸引用户的注意力。 $\bullet$  常用于对话框和窗体中的关键操作。

6. QDialogTextBox（对话框按钮框）：

$\bullet$  通常用于对话框，包含标准的对话框按钮，如"确定"、“取消"、“应用"等。 $\bullet$  它是一个包含多个按钮的容器，可用于管理对话框中的按钮。 $\bullet$  常用于对话框的底部以提供标准按钮。

这些Buttons控件通常与信号和槽机制一起使用，以响应用户的交互操作。你可以根据应用程序的需求选择适当的按钮类型，并通过设置它们的属性（文本、图标、状态等）来自定义按钮的外观和行为。Buttons控件是创建用户友好的图形用户界面的重要组成部分，可以用于实现各种操作和用户交互。

# 3.1.4项目视图组（基于模型）ItemViews(Model-Based)

ItemViews组件，它是一组用于显示和处理数据的模型驱动的控件。包含控件如下图所示：

![](assets/qt学习手册/381f117d905d313cf540cbb42ffd47185979343d91d1509ccb690e2ee647cae7.jpg)

1. QListView（列表视图）：

$\bullet$  QListView是一个用于显示列表或图标项目的控件，可以使用不同的视图模式。 $\bullet$  它可以与QAbstractListModel等数据模型一起使用，以显示自定义的数据。 $\bullet$  常用于文件浏览器、应用程序选项列表等。

2. QTreeView（树视图）：

$\bullet$  QTreeView用于显示树形数据结构，每个节点可以有子节点。 $\bullet$  它可以与QAbstractItemModel等模型一起使用，以构建树形结构。 $\bullet$  常用于文件系统浏览、目录结构、层次数据等。

3. QTableView（表格视图）：

$\bullet$  QTableView用于以表格形式显示数据，类似于电子表格。

- 它是一个灵活的控件，支持多列、多行的表格，并可以自定义每个单元格的内容。- 常用于数据库浏览、数据报表等。

4. QColumnView（列视图）：

- QColumnView 用于以列的形式显示项目，通常用于显示多列项目。- 它可以与 QAbstractItemModel 一起使用，以提供自定义的数据显示。- 常用于邮件客户端、资源管理器等。

5. QUndoView（撤销/重做视图）：

- QUndoView 用于显示撤销和重做操作的历史记录，允许用户浏览和撤销先前的操作。- 它通常与 QUndoStack 和 QUndoGroup 一起使用，以实现撤销和重做功能。- 常用于应用程序的编辑操作历史记录。

这些 ItemViews 控件是基于 Qt 的模型/视图架构构建的，其中数据由模型（通常是 QAbstractItemModel 的子类）提供，视图（如 QListView）用于显示和编辑数据。通过这种方式，你可以有效地管理和显示大量的数据，并支持各种数据的交互操作，如排序、过滤、编辑等。这些控件在创建复杂的数据驱动界面时非常有用，能够提供良好的用户体验和灵活性。

# 3.1.5 项目空间组（基于项）ItemWidgets(Item-Based)

用于显示和管理项目数据。这些控件通常被用于显示列表、表格、树等项目型的数据。如下图所示：

![](assets/qt学习手册/ff89a5f6d4ed24c2f91e6be17e973a3697b02ba1260142b0c6feabbfdf923ead.jpg)

1. QListWidget（列表控件）：

- QListWidget 是一个用于显示项目的控件，每个项目可以包含文本和图标。- 它适用于显示简单的项目列表，如应用程序选项或文件列表。- 用户可以单击项目以进行选择。

2. QTreeWidget（树控件）：

- QTreeWidget 用于以树形结构显示项目，每个项目可以包含子项目。- 它支持多列显示，每一列可以包含不同类型的数据。- 常用于显示层次化的数据，如文件系统浏览、目录结构等。

3. QTableWidget（表格控件）：

- QTableWidget 用于以表格形式显示项目，类似于电子表格。- 它支持多行、多列，并且每个单元格可以包含不同类型的数据。- 常用于显示和编辑二维表格数据，如数据库浏览、数据报表等。

这些基于项目的控件提供了一种直观的方式来显示和管理项目型数据，适用于各种应用程序，从简单的列表到复杂的数据图形。你可以选择合适的控件根据你的应用需求，然后使用模型/视图架构来管理数据的显示和交互。这些控件通常提供了丰富的API，允许你自定义项目的外观和行为，以满足特定的设计和用户需求。

9. DockWidget（QDockWidget）：

10. QAxWidget：

- QAxWidget 是用于嵌入 ActiveX 控件（eg:在 Windows 平台上嵌入 OLE 控件）的 Qt 控件。- 可以将 ActiveX 控件嵌入 Qt 应用程序中，以实现与外部组件的交互。

# 3.3 信号和槽

信号（Signal）：信号是Qt对象可以发出的事件或消息，通常与特定的事件相关联。例如，一个按钮对象可以有一个clicked信号，表示按钮被点击了。

槽（Slot）：槽是Qt对象的成员函数，用于响应信号。当信号发出时，与之相关联的槽函数将被调用，执行相关操作。

信号和槽都位于类中，不是  $\mathbb{C} + +$  标准代码。我们可以把槽函数绑定在某一个控件的信号上。当需要调用外部函数时，发送一个信号，此时与该信号相关联的槽便会被调用，槽其实就是一个函数，槽与信号的关联要由程序员来完成，关联方法有自动关联和手动关联。

# 3.3.1 自动关联

使用Qt信号和槽的自动关联，可加快开发速度，一般用于同一个窗体之间的控件关联，槽函数格式非常关键，格式为：

voidon_<窗口部件名称>_<信号名称>(<signal parameters>);

# 3.3.2 手动关联

信号和槽机制是QT的核心机制，要精通QT编程就必须对信号和槽有所了解。信号和槽是一种高级接口，应用于对象之间的通信，它是QT的核心特性，也是QT区别于其它工具包的重要地方。此外如果遇到不懂的函数或类，可以先选中，然后按F1键，即可查看介绍。

虽然Qt有自动关联功能，但涉及到多个窗体和复杂事件的时候，只能使用手动关联，手动关联使用connect这个函数。

connect(const QObject *sender, const char *signal, const QObject *receiver, const char *member, Qt::ConnectionType = Qt::AutoConnection);

通常只传递前四个参数，各个参数含义如下：

- sender: 发送对象；- singal: 发送对象里面的一个信号，格式一般为SIGNAL(信号);- receiver: 接收对象；- member: 接收对象里面的槽函数，格式一般为SLOT(信号)。- ConnectionType: 设置信号和槽的同步异步，一般使用默认值Qt::AutoConnection，可不填。connect(A, SIGNAL(B), C, SLOT(D));当对象A发出B信号时候，就会触发对象C的槽函数D

- 信号（signals）：是QT的关键字，而非  $\mathrm{C / C + + }$  的。signals关键字指出进入了信号声明区，随后即可声明自己的信号。- 槽（slots）：是普通的  $\mathrm{C + + }$  成员函数，当与其关联的信号被发射时，这个槽函数就会被调用。槽函数有的参数个数和类型，在对应的信号函数中必须一一对应，即信号函数的参数个数必须多于或等于槽函数的个数。- emit：是一个关键字，用于发出信号。当对象需要发出信号时，使用emit关键字来触发信号。下面是一个简单的示例，演示了信号、槽和emit的用法：

# 4.2.1 TCP 实现服务器和客户端

TCP 协议（Transmission Control Protocol）是一种面向连接的，可靠的，基于字节流的传输层通信协议，传输数据稳定可靠。在 Qt 中进行 TCP 网络编程的主要类是 QTcpSocket 和 QTcpServer。

在“帮助”索引中搜索到如下图所示两个重要类：

![](assets/qt学习手册/fda85211d95a714e8ddf2c0551e1a9f9d5631b5118f17d725a16b733d60d099c.jpg)

服务器编程中两个类都会用到，客户端编程中只会用到 QTcpSocket 对象。本实验中对 QTcpServer 类的基本使用：（1）监听客户端连接。（2）每当有新的客户端连接服务器的时候，会自动触发信号，（3）根据当前连接上的新的客户端创建一个 Socket 对象，将数据的收发动作交给 socket 套接字去处理。（4）关闭服务器 close(); 对 QTcpSocket 类的基本使用：（1）服务器端：有新连接时获取连接状态，绑定 socket。（2）客户端：通过 socket 连接服务器，连接成功触发信号。（3）当有数据到来时会触发信号，用 readAll() 读取。（4）通过读写 socket 收发数据。
